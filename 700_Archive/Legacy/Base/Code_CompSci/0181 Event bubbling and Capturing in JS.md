## Всплытие(bubbling)
Принцип всплытия очень простой. 
Когда с элементом взаимодействуют(событие), обработчики сначала чекают сам элемент, дальше его родителя, затем выше и выше,вверх по семейному древу.

*Почти все события всплывают.
Лишь* **focus** *не всплывает*

### event.target
Самый глубокий элемент называется - *целевым элементом*. Его можно чекнуть через **event target**.  

Отличие от **this(event.currentTarget**):
- **event.target** - это целевой элемент, на котором произошло событие.
- **this** - это текущий элемент,до которого мы успели всплыть, его щас обрабатывает.
#### Прекращение всплытия
Прекратить это можно через **event.stopPropagation**
Но,если честно,нет в этом пользы.
Проблема для которой это может понадобится можно решить по другому.
##### Погружение(Capture)
Погружение идет сверху вниз,догоняем эдемент.
Стандарт **DOM Events** описывает 3 фазы прохода события:
1. Фаза погружения(capturing phase) - событие идет сверху вниз.
2. Фаза цели(target phase) - событие дошло до целевого элемента.
3. Фаза всплытия (bubbling stage) - событие начинает всплывать.
