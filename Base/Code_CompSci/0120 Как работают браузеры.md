# Главная задача браузера
Главная задача браузера - это показывать веб ресурсы которые вы хотите, браузер берет их с сервера и покаывает в окне браузера. Локация браузера находится по URI(Uniform Resource Identifier).
## Структура браузера
1. Пользовательский интерфейс: Сюда входят адрессные строки, тулы всякие(поиск,закладки итд).
2. Двигатель браузера: Упорядичивает работу между UI(user interface) и рендер.
3. Рендер: отвечает за показ контента который мы хотим видеть. Например, контент который мы хотим увидеть это HTML or CSS , то рендерный двигатель парсирует это все и показывает нам уже запарсированную картинку.
4. Сеть: запрос HTTP,(200=ОК) связь с другими платформами и интерфейсами.
5. UI backend: отвечает за создание базовых фигур таких как коробки и окна, которые в свою очередь не зависят от железа или от браузера, общедоступный.
6. JavaScript interpreter: парсирует джавускрипт.
7. Хранилище данных: тут хранятся всякие данные которые нужны для того,чтобы открыть страницы. Данные типо кукис. Браузер также поддерживает механизмы хранилища такие как localStorage, IndexedDB, WebSQL and FileSystem.

**Парсинг 
	Парсинг это - преобразование текста в структуру которую может использовать код. Обычно запарсированный текст выглядит как дерево узлов которая отображает структуру текста. This is called a parse tree or a syntax tree.
	Например, парсированный 2+3-1 будет выглядить так:![[Pasted image 20230111144758.png]]
Грамматика
	Парс происходят по синтаксу,грамматике и вообще по законом кода который парсируется. Каждый формат который парсируется имеет свою грамматику и правила синтакса. It is called a ==context free grammar==. Человеческий язык не комп язык поэтому его нельзя запарсировать общепринятыми способами. 

**Методы парсировки
	Парсировку можно поделить на два процесса: lexical analysis and syntax analysis.
	Лексический анализ: 
		Лексический анализ это процесс разбиение входящих данных в токены. Токены это словарь язык:коллекция дейсвительных строительных блоков. Это как слова в словаре. Материал с которым мы строим предложения. Строительные блоки токена = Слова в словаре. 
	Анализ синтакса: 
		Это применение синтаксных правил языка.
	Работа парсера состоит из двух компонентов: Лексер(токенайзер) который отвествен за разбиение входящего в токены. И парсер который отвествен за создание дерева узлов по анализу структуры документа который оперирует с правилами синтакса языка. 
	Лексер умный,он отличает нужную инфу от хлама как белые поля,разрывы строк и тд 
	Весь этот процесс выглядит примерно так:![[Pasted image 20230111150530.png]]
	Парсер просит у лексера токены чтобы связать их с правилами синтакса. Если совпадает то этот токен идет в дерево и парсер просит другой токен. 
	А если нет, то парсер хранит токен у себя пока не найдет подходящий синтакс. А если нет значит документ не подходит и происходит ошибка.

	Верхний и нижний. Верхний определяет совпадают ли высоко структурный синтакс с правилами. А нижний обрабатывает входящие данные в синтакс.
