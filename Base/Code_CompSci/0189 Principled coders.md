# 7 deadly sins of coding
1. Use standards 
	Придерживаться общей,одной формы кода. Это можно сравнить с постоянной сменой шрифта. Текст можно прочитать, но экспериенс уже будет не таким приятным как мог бы быть.
2. Use principles
	S.O.L.I.D principles:
		1. S - Single responsibility
			 Это значит разделять код на блоки каждый из которых отвечают за разные задачи.
		2. O - Open/closed
			Это значит не изменять код прямо,а подходить с другой стороны,добавлять в пару другие строчки кода. Тогда будет модуль кода будет locked-in поэтому код не может быть разрушится при случаи чего.
		3. Liskov substitution 
			Это значит писать код который подходить друг другу,в одном стиле и тд.
		4. I - Interface segregation
			Это значит создавать мини блоки кода который работают как мост от фронтенда к бэкенду.
		5. D - Dependency Inversion
			Это значит не привязывать блоки кода напрямую с другими блоками кода. А заменить это более универсальными строчками кода.
3. Apply patterns 
Паттерны в кодинге чтобы было легче работать(ресерч)
4. Adequate names
Use this rules to:
- avoid encodings
- expand abreviatiions
- use clear distinctions 
- no "magic" value. показывай откуда они появились 
5. Test your code more often
6. Realistic Time management
double or tripple the time you need to project.
7. Do not rush. Take your time
Не торопясь построй хороший фундамент. Когда ты будешь ошибаться ты будешь начинать не сначала, а с фундамента.
