# Что Реакт?
Реакт это фреймворк для джиэс, нужен для создания юзер интерфейса. 
## Зачем Реакт?
- мгновенно загружает страницы
- обрабатывает большие объемы динамического контента, который меняется во время пользовательского просмотра
- дает возможность писать выразительный инкапсулированный код ([[201 Инкапсуляция]]), пригодный для повторного использования

#### Почему Реакт - Реакт?
1. Виртуальный DOM (Virtual DOM): Реакт использует виртуальный DOM для управления обновлением элементов на веб-странице. Вместо того, чтобы непосредственно изменять реальный DOM при каждом изменении состояния, Реакт создает виртуальное представление DOM и сравнивает его с текущим состоянием. Затем, он определяет минимальное количество изменений, которые необходимо внести в реальный DOM, и применяет их. Это позволяет снизить количество операций обновления DOM, что делает Реакт более эффективным и быстрым.
    
2. Эффективное управление состоянием: Реакт предлагает эффективное управление состоянием при помощи хуков (hooks), таких как useState и useEffect. Хуки позволяют разработчикам управлять состоянием компонента и оптимизировать его обновление. Реакт также поддерживает рендеринг только изменений (shouldComponentUpdate или React.memo), что позволяет избегать ненужных перерисовок компонентов и улучшает производительность.
    
3. Компонентная архитектура: Реакт основан на компонентной архитектуре, что позволяет разбить веб-приложение на небольшие и переиспользуемые компоненты. Это способствует лучшей структуризации кода и повторному использованию, что в свою очередь может улучшить производительность и общую скорость работы приложения.
    
4. Возможность использования виртуального списка (Virtual List): Реакт предлагает возможность использования виртуального списка, который отображает только видимую часть данных на странице, что значительно увеличивает производительность при работе с большими объемами данных.
    
5. Оптимизация при сборке: Реакт предлагает инструменты для оптимизации и минификации кода при сборке проекта. Например, можно использовать инструменты, такие как Babel и Webpack, для сжатия и объединения файлов JavaScript, что может улучшить скорость загрузки и выполнения приложения.
