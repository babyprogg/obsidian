# Базовое понятие
Hoisting - это продвижения функции,переменной и тд вверх по важности. Неважно где была обьявлена функция и тд , она все равно будет компилится первой.
## Для чего?
Для корректной работы с JS. Мы можем  вызвать функцию,даже если она была объявлена позже. А вот с переменными так лучше не делать.
### Как использовать?
1. Не вызывайте переменные и функции до того,как объявите их. Сначала объявите переменные, а потом уже вызывайте их.
2. let и const поднимаются, но мы не можем получить к ним доступ до их объявления из за Временной мертвой зоны(temporal dead zone)
